# 机伴管理 - 新增/编辑功能修复

## 任务
修复机伴管理页面的新增和编辑功能

## 问题分析

### 根本原因
前端 Companions.tsx 调用了不存在的后端路由：
- `POST /admin/companion-models` - 后端无此路由
- `PUT /admin/companion-models/:id` - 路径错误，应为 `/db/companion-models/:id`
- `DELETE /admin/companion-models/:id` - 路径错误且使用 DELETE 方法而非 POST

### 后端实际路由（db.js）
```
GET    /db/companion-models         # 获取列表 ✅
PUT    /db/companion-models/:id     # 更新机伴 ✅
POST   /db/companion-models        # ❌ 不存在
```

### 后端通用 API（admin.js）
```
POST   /admin/insert   # 通用插入（table + rows）
POST   /admin/delete  # 通用删除（table + filters）
POST   /admin/update  # 通用更新（table + data + filters）
```

## 修复方案

### Companions.tsx 改动

| 操作 | 原调用 | 新调用 |
|------|--------|--------|
| 获取列表 | `GET /db/companion-models` | 不变 ✅ |
| 新增 | `POST /admin/companion-models` | `POST /admin/insert` + `table: companion_models` |
| 编辑 | `PUT /admin/companion-models/:id` | `PUT /db/companion-models/:id` |
| 删除 | `DELETE /admin/companion-models/:id` | `POST /admin/delete` + `table + filters` |

### 新增功能
- 新增/编辑弹窗表单
- 基本信息：名称、模型编码、语音ID、MBTI、描述
- 性格设定：初始形象、核心命题、性格特质

### UI 改进
- 从表格改为卡片列表
- 显示属性标签（初始形象、MBTI、核心命题）
- 编辑/删除按钮
