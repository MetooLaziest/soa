# 小游戏素材管理页面 - 第2步完成

## 目标
在 https://soa.laziestlife.com/admin 后台管理系统中新增「小游戏素材管理」页面，支持上传/预览/删除游戏贴图素材。

## 完成内容

### 1. 前端 - MiniGameAssets.tsx（新建）
- **路径**: `iot-ai-doll-frontend/src/pages/admin/MiniGameAssets.tsx`
- **功能**:
  - 12个游戏图标概览卡片（100层/消消乐/猜拳/记忆卡牌/FlappyBird/天降好礼/找不同/钓鱼/塔楼建筑/平台跳跃/吃豆人/煎鱼）
  - 拖拽上传区 + 点击上传按钮
  - 素材网格展示（缩略图 + 文件名 + 大小）
  - 点击放大预览（全屏弹窗）
  - 删除确认
  - 统计信息（总数 + 总大小）

### 2. 前端 - Layout.tsx（修改）
- 新增菜单项：`{ key: 'minigame-assets', label: '小游戏素材管理', icon: '🎮', path: '/admin/minigame-assets' }`

### 3. 前端 - App.tsx（修改）
- 新增路由：`<Route path="minigame-assets" element={<MiniGameAssets />} />`
- 新增 import：`import MiniGameAssets from './pages/admin/MiniGameAssets'`

### 4. 后端 - game-assets.js（新建）
- **路径**: `/var/www/iot-ai-doll/backend/src/routes/game-assets.js`
- **API**:
  - `GET /api/game-assets` — 列出 game-assets 目录所有文件（名称、URL、大小、修改时间）
  - `POST /api/game-assets/upload` — multer 单文件上传（PNG/JPG/GIF/WebP/SVG，最大5MB）
  - `DELETE /api/game-assets/:name` — 删除文件（含路径穿越防护）
- **存储目录**: `/var/www/iot-ai-doll/frontend/dist/epet1/game-assets/`

### 5. 后端 - index.js（修改）
- 新增 import 和路由注册

### 6. Nginx 配置（修改）
- 新增 location `/epet1/game-assets/` 静态文件映射（7天缓存）

### 7. 部署
- 前端构建并部署到 `/var/www/iot-ai-doll/frontend/dist/`
- PM2 iot-backend 重启成功
- Nginx reload 成功

## 验证结果
- ✅ API `GET /api/game-assets` 返回 `{"ok":true,"count":0,"assets":[]}`
- ✅ 后端语法检查通过（Node.js v18 ES Module）
- ✅ Nginx 配置测试通过
- ✅ PM2 iot-backend 运行正常

## 遇到的问题及解决
- **问题**: 初始上传的 game-assets.js 包含 TypeScript 类型注解 (`err: any`)，Node.js v18 不支持
- **解决**: 重写为纯 JavaScript，移除所有类型注解，模板字符串改为字符串拼接
