# E-Pet 1 设计需求文档 & 产品玩法说明书

> 供设计师输出视觉稿 / 供团队内部宣讲产品玩法

---

## 一、产品一句话介绍

**「扫描实体玩偶，与爱宠共度庭院时光」**

一款 NFC 实体联动 + AI 情感陪伴的养成游戏。
买一只内置 NFC 的玩偶，扫描绑定后在手机上养育它——
抚摸、喂食、听它说话、送它去旅行，收集它从全世界带回的明信片。

---

## 二、核心体验关键词

| 关键词 | 含义 |
|--------|------|
| **治愈感** | 宠物萌态、柔和配色、轻柔动效、无压力养成 |
| **收集欲** | 6只宠物 + N张明信片 + 碎片合成 |
| **情感羁绊** | AI 对话有记忆，宠物"认识"玩家 |
| **现实联动** | NFC 实体玩偶，线上线下互通 |
| **期待感** | 宠物出游 12 小时，回来时翻牌拆明信片 |

---

## 三、目标用户画像

- **主要群体：** 18-30 岁女性（轻度游戏用户）
- **次要群体：** 二次元爱好者、收集癖玩家
- **核心诉求：** 情感陪伴、无肝游戏、社交货币（明信片/宠物展示）

## 四、视觉风格定义

### 4.1 整体风格
**日系治愈风 + 轻量扁平插画**

- 参考方向：《旅行青蛙》《角落生物》《猫和老鼠》暖调插画
- 不走写实风，走可爱圆润路线
- 背景有景深层次（近景/中景/远景），不单调

### 4.2 色彩体系

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色（Primary） | `#7EC8A4` | 薄荷绿，活力又安静 |
| 辅助色（Secondary） | `#F5C76D` | 暖黄色，代表情绪值/货币 |
| 强调色（Accent） | `#F07E54` | 珊瑚橙，用于按钮/通知 |
| 背景色（Background） | `#FFF8F0` | 米白，温暖不刺眼 |
| 文字色（Text） | `#4A4A4A` | 柔和深灰 |
| 卡片色（Card） | `#FFFFFF` | 纯白 |
| 庭院背景渐变 | `#E8F5E9` → `#FFF8F0` | 草坪绿到米白 |

> 以上为主场景配色，各子页面可在主色体系内适当延伸。

### 4.3 宠物视觉规范

| 要求 | 说明 |
|------|------|
| 体型 | 圆润可爱，头身比约 1:1（婴儿感） |
| 尺寸 | 正视图为主，大眼无脸型 |
| 动作帧 | 每种状态至少 3 帧（idle / happy / eating / sleeping / traveling / greeting） |
| 输出格式 | PNG 透明底 + SVG 矢量源文件 |
| 分辨率 | 建议 512×512px 起步，导出 2x / 3x |

### 4.4 明信片视觉规范

| 要求 | 说明 |
|------|------|
| 比例 | 3:2（横向风景为主） |
| 风格 | 插画风，不是照片风 |
| 构图 | 宠物为主角，场景为背景 |
| 配套动画 | 每张明信片配一个 3-5 秒的循环微动画（MP4/GIF/Lottie） |
| 稀有度视觉区分 | N=白底 / R=绿框 / SR=蓝框 / SSR=金框 / UR=彩虹全息 |

## 五、主要场景设计需求

### 场景 1：庭院主页（最核心页面）

**设计师输出：庭院主视觉大图**

```
┌──────────────────────────────────┐
│  [😌 情绪值]              [⚙️]  │  ← 左上角情绪值徽章，右上角设置
│                                  │
│     ☁️        ☁️                 │  ← 天空层（装饰性云朵飘动）
│                                  │
│    🌿    [宠物A]  [宠物B]   🌿   │  ← 宠物层（宠物 idle 动画）
│                                  │
│         ┌──────────┐             │
│         │   桌子   │  💐        │  ← 前景层：喂食桌 + 装饰花草
│         └──────────┘             │
├──────────────────────────────────┤
│ [其他]                           │
│                      [漂流瓶]    │
│                      [藏品库]    │
│                      [明信片]    │
│                      [小游戏]    │
│                      [商店]      │  ← 右下角模块入口（竖排图标+标签）
└──────────────────────────────────┘
```

**设计要点：**
- 整体为竖屏全屏背景图（IPhone 13 / 主流竖屏尺寸）
- 庭院有层次感：天空（远）→ 宠物（中）→ 桌子/前景（近）
- 宠物以 3D 软萌渲染风格呈现，idle 时轻微上下浮动
- 两宠物靠近时有"对视"或"蹭头"的微互动动画
- 右下角 5 个图标需要明显可点击（带投影/描边）

---

### 场景 2：宠物互动页

**进入方式：** 点击庭院中的宠物

```
┌──────────────────────────────────┐
│  [←]                   [✈️出游] │
│                                  │
│         [宠物全身大图]            │  ← 宠物居中，约占屏幕 60%
│         (可交互区域划分)          │  ← 头/身/背/尾 区域标注
│                                  │
├──────────────────────────────────┤
│  [🍖 摸头] [🤚 摸身] [💆 摸背]   │  ← 抚摸按钮栏
│       [💬 对话输入框...]         │
│       [发送]                     │
└──────────────────────────────────┘
```

**设计要点：**
- 宠物大图有呼吸/微动效果
- 抚摸不同部位 → 宠物有对应反馈动画（头被摸=蹭/身被摸=蹭滚/背=伸展）
- AI 对话气泡从宠物方向飘出，带小尾巴
- 出游按钮有明显引导感（飞机图标 + "送它去旅行"）

### 场景 3：宠物出游 & 明信片翻牌

**出游确认弹窗：**
```
┌──────────────────────────────────┐
│       ✈️ 准备出发！               │
│                                  │
│     [宠物图]  要离开 12 小时       │
│                                  │
│   目的地：[随机/可选？]            │
│                                  │
│   [暂不出发]      [确认出发 ✓]    │
└──────────────────────────────────┘
```

**明信片翻牌（宠物回来时）：**
```
┌──────────────────────────────────┐
│                                  │
│    12 小时后 · 宠物回来啦！🎉     │
│                                  │
│    ┌────────────────────┐        │
│    │                    │        │
│    │    [明信片正面]     │        │  ← 点击翻牌动画
│    │    （插画+宠物）    │        │
│    │                    │        │
│    └────────────────────┘        │
│                                  │
│    [明信片名称]  ✦✦✦ SSR         │
│    "在巴黎铁塔下看日落"           │
│                                  │
│    [存入收藏 ✓]                  │
└──────────────────────────────────┘
```

**设计要点：**
- 翻牌动画：卡牌从背后翻转到正面，有轻微弹性
- 稀有度越高，翻牌停留时间越长（SSR/UR 有特写光效）
- 重复明信片 → 显示"已拥有 ✕3" 灰化效果

## 六、动效设计规范

| 场景 | 动效描述 | 时长 |
|------|---------|------|
| 宠物 idle | 轻微上下浮动 + 眨眼 | 循环 3s |
| 宠物被摸 | 对应部位压缩+回弹 + 心形飘出 | 0.6s |
| 宠物进食 | 低头啃食 + 尾巴摇 | 2.5s |
| 宠物睡眠 | 缓慢呼吸起伏 | 循环 |
| 宠物出游离开 | 向右滑出画面 | 1.2s |
| 宠物回来 | 从门外探头 → 跑向中央 | 1.5s |
| 明信片翻牌 | Y轴翻转 180° + 弹性停止 | 0.8s |
| 情绪值+ | 数字向上飘出+消失 | 0.6s |
| 漂流瓶投放 | 瓶子下沉消失 | 0.5s |
| 碎片获得 | 碎片飞入背包 + 光效 | 1s |

## 八、产品玩法简介（PPT 宣讲用）

> 建议 PPT 共 10-12 页，以下为每页核心文案

**第 1 页：封面**
> E-Pet 1
> 你的 NFC 电子宠物
> 旅行青蛙 × AI 情感陪伴

**第 2 页：我们卖的不只是游戏**
> 一只实体玩偶 + 一个线上宠物
> NFC 一扫，爱宠即刻"活"过来
> （配玩偶+手机联动示意）

**第 3 页：核心循环**
> 扫描绑定 → 庭院养育 → 互动陪伴 → 出游收集 → 情绪值消费
> （配核心循环图）

**第 4 页：如何获得宠物**
> 买一只内置 NFC 的玩偶
> 用手机轻轻一碰
> 它就成为你的专属宠物
> （NFC 扫描示意配图）

**第 5 页：庭院时光**
> 最多同时养育 2 只宠物
> 它们会在庭院里自己玩耍、互动
> 轻触它们，开始互动
> （庭院场景配图）

**第 6 页：情感互动**
> 摸头、摸背、摸肚子——AI 会记得
> 对话——宠物记得你说过的话
> 喂食——用情绪值购买食物投喂
> （互动页配图）

**第 7 页：送它去旅行**
> 安排宠物出游，12 小时后回来
> 它会从全世界带回一张明信片
> 不同稀有度，不同惊喜
> （翻牌动画配图）

**第 8 页：收集系统**
> 6 只宠物，每只都有成长线
> 20+ 张明信片，持续扩充
> 碎片可提升宠物成长度，解锁商店限定物品
> （藏品库+明信片墙配图）

**第 9 页：情绪值经济**
> 每日抚摸、收到漂流瓶、参与小游戏
> → 获得情绪值
> 情绪值 → 购买虚拟食物/商店限定商品
> （经济闭环图）

**第 10 页：漂流瓶社交**
> 发送你想说的话，漂流到远方
> 随机被其他玩家捡到
> 偶尔收到情绪值奖励或宠物碎片
> （漂流瓶界面配图）

**第 11 页：开发计划**
> v1（MVP）：基础养成 + 出游 + 藏品库
> v2：漂流瓶 + 小游戏
> v3：实物商城 + 微信支付 + 小程序/APP
> （roadmap 图）

**第 12 页：愿景**
> 让每一只玩偶都有自己的故事
> E-Pet 1 · 即将上线
