# E-Pet1 Debug Session — 2026-06-02 15:00

## 目标
修复 E-Pet1 React+PixiJS 项目"宠物精灵不显示"问题。

## 排查结果

### API 响应分析
- `/api/epet/monsters` 返回 `{ success: true, monsters: [...] }`，列表项中**无 fullness 字段**，stats 嵌套在 `stats` 对象内
- `/api/epet/:id` 单宠详情有 `fullness`，但被 `{ success: true, pet }` 包装

### 发现的问题
1. **Canvas 尺寸不匹配** — `AREA_W=380, AREA_H=260` 但 CSS 设 `100vw×100vh`，精灵坐标严重偏离可视区域
2. **API 类型不一致** — `MonsterApiItem` 中 stats 嵌套，前端 Pet 类型期望扁平字段
3. **`fetchPet` 未解包** — 直接返回 data.pet，但调用方可能期望直接用

## 修复内容

### 1. epet.ts — 类型修复
- 新增 `MonsterApiItem` 和 `Stats` 接口对齐 API 响应结构
- 新增 `transformMonsterItem()` 将嵌套 stats 展平为 Pet 字段
- `fetchPet` 正确解包 `data.pet`

### 2. Game.ts — 全屏 Canvas + 视口自适应
- `getViewport()` 动态获取 `window.innerWidth/Height`
- Canvas 初始化尺寸 = 视口实际尺寸（不再硬编码 380×260）
- 椭圆轨道 `baseRx/baseRy` 基于视口比例计算
- 精灵尺寸 `spriteW = Math.min(100, W * 0.15)` 响应式
- 左右翻转根据椭圆角度判断
