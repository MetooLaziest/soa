# 全站页面样式统一修复

## 目标
用户反馈各页面导航栏高度、Hero区间距不一致，要求统一修复，同时修复产品发布子页面（product-P001~P018）的样式并对齐产品发布页。

## 问题诊断
1. **导航栏高度不一致**：index.html 有滚动JS动态改变 navbar padding（10px vs 15px），部分页面有内联 navbar CSS 覆盖
2. **Hero区域不统一**：
   - products/works/team/contact 页用 `.{page}-hero` 类，padding `80px 20px 50px`
   - productsfuture 页用 `.products-hero`（已从 `.launch-hero` 改过）
   - product-P 详情页用 `.product-detail-hero`，padding 仅 `60px 20px`
   - about 页用 `.hero-gradient`
   - 各页面 h1 字号不一致（2.2em vs 2.5em）
3. **产品详情子页面问题**：
   - Hero padding 太小（60px vs 80px）
   - 有 `← 返回产品池` 链接
   - 大量内联 CSS（.detail-card, .status-badge 等）与 global.css 重复
   - body padding-top 内联覆盖
4. **body padding-top 不一致**：about.html 用 25px，部分页面用 56px，部分依赖 global.css
5. **index.html** 有旧的被注释掉的 header/nav 和 `.hero` banner 区

## 解决方案
1. 创建统一类名 `.page-hero`，在 global.css 中定义（与 .hero-gradient 并列）
2. 所有子页面（除首页 banner 外）统一使用 `.page-hero`
3. 产品详情页：移除所有内联CSS，删除 nav-back 链接，hero 对齐
4. index.html：删除旧 nav/header 残留，删除滚动 JS（navbar padding 变化），删除 沃兹基 slogan
5. about.html：修正 padding-top 从 25px 改为依赖 global.css 的 56px

## 修改的文件（26个HTML + 1个CSS）
- global.css: 新增 .page-hero, .product-detail-content 样式
- index.html, about.html, works.html, products.html, productsfuture.html, team.html, contact.html, 50x.html, clayworks.html
- product-P001~P018.html（18个产品详情页）
