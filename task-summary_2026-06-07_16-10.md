# Admin 前端重构 + Pets 实体管理迁移

## 目标
将 Admin 前端 CompanionEdit/CompanionList 从「管理型号模板(model_id)」改为「直接管理宠物实体(nfc)的 system_prompt」，实现人设分层编辑。

## 架构确认
- **pets.nfc** = 宠物实体ID（主键）
- **pets.model_id** = 对应型号模板（外键）
- **pets.system_prompt** = 实体级提示词覆盖（优先于 models.system_prompt）
- **pets.temperature** = 无此字段，温度沿用 models.temperature
- **pet_rag_kbs** = 宠物-RAG关联表（pet_nfc ↔ rag_kb_id）

## 完成的工作

1. **admin-pets.js** — 完整 CRUD 路由（含 GET/:nfc/rags、POST/DELETE /:nfc/rags/:ragId）
2. **index.js** — 确认 admin-pets.js 和 admin-models.js 已注册
3. PM2 重启成功

### 前端
1. **CompanionList.tsx** — 改为展示 pets 实体（nfc、display_name、model_name、system_prompt预览）
2. **CompanionEdit.tsx** — 4层编辑器（L1-L4），保存到 pets.system_prompt；temperature 显示来自 model（不可编辑）；对话测试 /api/epet/:nfc/chat
3. **CompanionRAG.tsx** — 改为使用 /api/admin/pets/:nfc/rags

5. **App.tsx** — basename 从 `/admin2` 改为 `/admin`
