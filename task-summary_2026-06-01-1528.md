# E-Pet 页面"加载中"卡死修复

## 时间
2026-06-01 15:28 - 15:32

## 目标
修复 https://soa.laziestlife.com/epet/?id=9527 页面显示"加载中"、无宠物卡片的问题

## 根因分析
- **直接原因**：index.html 第654-657行 `renderPetCards()` 函数中的 onclick 属性存在 JavaScript 语法错误
- **错误代码**：`doFeed(\'' + p.petId + '\')` — 单引号字符串内的 `\'` 转义导致解析器混淆
- **影响范围**：整个 `<script>` 块因 SyntaxError 无法解析 → 所有JS不执行 → init()/loadVisiblePets()/updateUI() 全部不运行
- **问题来源**：之前通过 backup_dist 恢复文件时引入了有此bug的版本

## 修复操作
1. 从服务器下载 `/var/www/iot-ai-doll/frontend/dist/epet/index.html` 到本地 `/tmp/epet-current.html`
2. 用 Python 脚本修复4处引号转义：
   - `doFeed(\'' + p.petId + '\')` → `doFeed(' + p.petId + ')`
   - `doPet(\'' + p.petId + '\')` → `doPet(' + p.petId + ')`
   - `doClean(\'' + p.petId + '\')` → `doClean(' + p.petId + ')`
   - `openChat(\'' + p.petId + '\')` → `openChat(' + p.petId + ')`
3. `node --check` 验证JS语法通过（EXIT 0）
4. SCP上传到服务器，验证HTTP 200且修复内容生效

## 验证结果
- curl 返回 HTTP 200, 29228 bytes
- 修复后的代码 `openChat(' + p.petId + ')` 确认存在于部署文件中
