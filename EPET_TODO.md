## 🔴 P0 — 必须修复

### 1. 家具像素碰撞 Y 轴映射 bug
- **现象**: 伞上方透明区域仍阻挡通行，伞柄区域无碰撞
- **根因**: sprite collision 像素数据与 PixiJS 渲染坐标的 Y 轴映射不一致，垂直翻转后仍有偏差
- **文件**: `epet1/src/game/CollisionMap.ts`, `epet1/src/game/Game.ts`
- **状态**: 多次修复未解决，需深入 debug

### 2. 用户注册/登录系统
- **现状**: `DEMO_USER_ID=2` 硬编码，后端只有游客模式(guest_前缀) + 微信 createOrUpdate
- **需要**: 
  - 手机号注册/登录（微信授权或其他方式）
  - Token 鉴权中间件
  - 前端登录页面 + 用户信息持久化
- **关联**: API 无鉴权问题(P1)与此绑定

### 3. API 鉴权 + 多用户数据隔离
- **现状**: 所有 API 无鉴权，任何人可传任意 user_id 操作
- **需要**: JWT/Session 中间件，所有 /api/epet1/* 路由加用户校验
- **状态**: 升级为 P1 → P0（与用户系统一起做）

### 4. 出游归来领取流程
- **现状**: 后端 `/travel/:id/claim` 已写，前端有倒计时但到 0 后只显示"归来时通知你"，无自动检测+领取 UI
- **需要**:
  - 轮询/心跳检测旅行归来状态
  - 归来弹窗 + 明信片展示 + 情绪值奖励动画
  - 明信片 `is_new` 红点提示（后端已有字段，前端未用）
- **文件**: `epet1/src/App.tsx` (TravelModal / PostcardModal)
