## 问题根因分析

### 问题1：图片反转
**根本原因**：前端 `App.tsx` 中 `PET_IMAGES` 映射写死了错误的对应关系：
```
错误：model_id=1 → pet-white.png, model_id=2 → pet-blue.png
正确：model_id=1(海浪沫沫) → pet-blue.png, model_id=2(团子糯糯) → pet-white.png
```
数据库 models 表的 `image_url` 字段也有同样错误。后端 `epet.js` 的 `MONSTER_TYPES` 也被之前多次 sed 操作损坏。

### 问题2：触摸进不了互动页面
**根本原因**：之前修复 MONSTER_TYPES 时用只有 61 行的最小版本**覆盖了完整的 527 行 epet.js**，丢失了 feed/pet/clean/monsters/chat 等所有路由，导致前端调用这些 API 时全部失败。

## 经验教训
1. **不要用最小代码覆盖完整路由文件** — 之前为了修复 MONSTER_TYPES 把 527 行截断到 61 行，丢失大量 API
2. **sed 多行替换不可靠** — 应该用 Python 读取→修改→base64 写回的方式
3. **SCP 上传目录会创建子目录** — `scp.put(dist, remote)` 会把 dist 作为子目录放入，需要逐文件上传或事后移动
4. **前后端映射必须一致** — 数据库、后端 MONSTER_TYPES、前端 PET_IMAGES 三处必须同步
