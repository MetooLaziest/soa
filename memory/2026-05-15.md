#### 用户反馈

#### 排查过程

   - bath → command: "start" ✅

   ```json
   {"logId":24,"deviceType":"bath","deviceGroup":"all","command":"start"}
   ```

   - 固件定义: `{"bath", "bathroom", "on", "BAT/BAT/OPEN\n"}`
   - 云端下发: `command: "start"`
   - 结果: 匹配失败 → 默认关闭

固件端 `serialCmdTable[]` 中的 command 字段与云端 API 验证规则不一致：

### 16:31 - ✅ 固件/云端指令格式不匹配问题定位

#### 用户反馈
ESP32 收到扫地机 (vacuum) 和浴缸 (bath) 指令时显示【关闭】，实际应该执行 start。

#### 排查过程
1. ✅ 检查控制日志 - 日志内容正确：
   - bath → command: "start" ✅
   - vacuum → command: "start"/"stop" ✅

2. ✅ 检查云端下发格式 - 正确：
   ```json
   {"logId":24,"deviceType":"bath","deviceGroup":"all","command":"start"}
   ```

3. ❌ 发现问题：**固件串口指令映射表不匹配**
   - 固件定义: `{"bath", "bathroom", "on", "BAT/BAT/OPEN\n"}`
   - 云端下发: `command: "start"`
   - 结果: 匹配失败 → 默认关闭

#### 问题根因
固件端 `serialCmdTable[]` 中的 command 字段与云端 API 验证规则不一致：
- 云端: bath 只支持 `start`
- 固件: bath 映射的是 `on`/`off`

#### 解决方案
给中控板团队提供技术说明文档，要求固件修改 `serialCmdTable[]` 匹配云端指令格式。

#### 正确的指令映射表
| deviceType | 云端 command | 固件需匹配 |
|------------|--------------|-------------|
| bath | start | "start" |
| vacuum | start/stop | "start"/"stop" |
| light | on/off | "on"/"off" |
| curtain | on/off | "on"/"off" |
| purifier | on/off | "on"/"off" |
| tv | on/off | "on"/"off" |
