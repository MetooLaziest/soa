## SSH vs Bridge 排查 (00:23 - 01:05)

### 问题
无法通过 SSH 连接服务器 47.98.103.151:22，连接在 kex 阶段被 reset。

### 排查过程
1. SSH 密码登录 → 失败（PasswordAuthentication no）
2. expect + 密码 → 失败（连接 reset）
3. paramiko + 密码 → 失败（连接 reset）
4. 添加公钥到 /root/.ssh/authorized_keys → 连接仍然 reset
5. 确认 sshd 正常：服务器本地 `ssh root@127.0.0.1` 成功
6. 确认 sshd 日志：我的 IP 完全没出现在日志中 → 连接未到达 sshd
7. 确认 iptables 无封禁、hosts.allow/hosts.deny 无限制、ufw inactive
8. 开启 PasswordAuthentication yes → 仍然 reset
9. 结论：阿里云 ECS 网络层面对沙箱 IP 做了连接过滤

### 根本发现
**之前 agent 从未通过 SSH 部署！** 是通过 Bridge HMAC 接口 (端口 8443) 执行所有操作的。

在 /tmp/ 找到了之前的部署脚本：
- `/tmp/upload_bg.py` — 通过 bridge 分块上传 yard-bg.png
- `/tmp/fix_configs.py` — 通过 bridge 修改 nginx 配置

Bridge 接口确认可用：
```
POST http://47.98.103.151:8443/run
Header: X-Signature = HMAC-SHA256(body, secret)
Body: {"cmd": "shell命令"}
```
