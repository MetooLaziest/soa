## IoT AI 玩偶控制项目 - 前端登录问题调试

### 项目信息
- **域名**: https://soa.laziestlife.com
- **后端服务器**: 47.98.103.151:3000 (阿里云轻量应用服务器)
- **SSH**: root@47.98.103.151, 密码: 123ASDasd!@#
- **前端仓库**: /Users/lintingrui/.qclaw/workspace/iot-ai-doll-frontend

### 问题诊断
用户反馈：登录后跳转到主页，但随即又跳回登录页

### 发现的问题

1. **混合内容问题 (Mixed Content)**
   - 网站通过 HTTPS (soa.laziestlife.com) 访问
   - 但 API 请求发送到 HTTP (http://47.98.103.151:3000/api)
   - 浏览器会阻止 HTTPS 页面中的 HTTP 请求

2. **API_BASE 硬编码 IP 地址**
   - `src/api/client.ts`: API_BASE = 'http://47.98.103.151:3000/api'
   - `src/api/chat.ts`: API_BASE = 'http://47.98.103.151:3000/api'
   - `vite.config.ts`: proxy target = 'http://47.98.103.151:3000'

3. **authStore loading 状态问题**
   - `init()` 方法设置 `loading: false`
   - 但 `login()` 方法没有设置 `loading` 状态
   - 可能导致状态不同步

### 修复方案

1. **修改 API_BASE 为相对路径**
   - 改为 `/api` (使用 Vite proxy 开发，Nginx proxy 生产)
   
2. **修改 vite.config.ts proxy**
   - target 改为 `http://localhost:3000` (服务器本地)
   - 或保持 `http://47.98.103.151:3000` 但仅在开发时使用

3. **Nginx 配置反向代理**
   - 生产环境通过 Nginx 代理 /api 到后端服务
   - 避免 HTTPS → HTTP 的混合内容问题

### 已执行的修复

1. 修改 `src/api/client.ts`: API_BASE = '/api'
2. 修改 `src/api/chat.ts`: API_BASE = '/api'
3. 添加 authStore.login() 中设置 `loading: false`
4. 修改 vite.config.ts proxy target

## 2026-05-18 15:50 - 机伴管理增删改功能修复

### 问题
用户反馈：机伴管理页面无法新增和编辑

### 根因分析

| 操作 | 前端原调用 | 后端实际路由 | 结果 |
|------|-----------|-------------|------|
| 获取列表 | `GET /db/companion-models` | ✅ 存在 | 正常 |
| 新增 | `POST /admin/companion-models` | ❌ 无此路由 | 失败 |
| 编辑 | `PUT /admin/companion-models/:id` | ✅ `/db/companion-models/:id` | 路径错误 |
| 删除 | `DELETE /admin/companion-models/:id` | ❌ 后端只支持 POST | 方法错误 |

### 修复方案

| 操作 | 修复后调用 |
|------|-----------|
| 新增 | `POST /admin/insert` + body: `{ table: 'companion_models', rows: [...] }` |
| 编辑 | `PUT /db/companion-models/:id` |
| 删除 | `POST /admin/delete` + body: `{ table: 'companion_models', filters: { id } }` |

### 后端通用 API（admin.js）

```javascript
POST /admin/insert   // 通用插入 { table, rows }
POST /admin/update   // 通用更新 { table, data, filters }
POST /admin/delete   // 通用删除 { table, filters }
```
