### 18:35 - DeviceBindPage Response 解析 Bug 修复

- 用户绑定设备时报"设备SN不存在"错误
- 根因：`/api/db/query?...&single=true` 返回的是直接对象 `{"id":"...","sn":"MIAODA-001"}`（不是 `{rows:[...]}` 格式）
  - `const rows = Array.isArray(regData) ? regData : (regData.rows || [])` → `rows = []`（因为对象没有 `.rows` 属性）

- 修复：去掉 `&single=true`，后端统一返回 `{rows: [...]}` 格式
- 同时新增统一错误处理 `apiFetch()` helper

### 18:46 - bind-full 500 错误修复（PostgreSQL array 类型问题）
