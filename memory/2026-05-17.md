## 22:34-22:45 - 根因确认与修复

### 根因：`Failed to fetch` 是混合内容错误

前端 JS 构建产物中硬编码了 `http://47.98.103.151/api`（HTTP 协议），而页面通过 `https://soa.laziestlife.com` 加载。浏览器**阻止了 HTTPS 页面发起的 HTTP 请求**（混合内容策略），所以报 `Failed to fetch`。

### 排查过程

1. **CORS 检查** - `OPTIONS` 预检请求返回 204，`Access-Control-Allow-Origin: *`，CORS 正常
2. **前端 JS 检查** - 在 `index-CYLpB92E.js` 中发现硬编码的 `http://47.98.103.151/api`
3. **grep 确认** - 仅这一个 JS 文件包含该 IP 地址

### 修复

```bash
# 替换 JS 文件中的 API URL
sed -i 's|http://47.98.103.151/api|/api|g' /var/www/iot-ai-doll/frontend/dist/assets/index-CYLpB92E.js
```

验证：grep 返回 0，IP 地址已全部替换。

### 密码测试

用 `admin123` 测试登录 API，仍然返回"用户名或密码错误"。说明数据库中的密码哈希不是 `admin123` 生成的。

### 待完成
- [ ] 重置用户 `15622205445` 的密码为 `admin123`
- [ ] 验证前端登录功能恢复正常

### 教训
- 前端构建时 `VITE_API_URL` 必须配置为相对路径（`/api`）或 HTTPS 域名，不能使用 HTTP IP 地址
- 混合内容错误在浏览器控制台不会显示明确的 CORS 错误，而是直接 `Failed to fetch`
