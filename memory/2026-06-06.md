## Admin2 前端部署完整调试过程 (23:26-00:04)

### 阶段1: HTTPS 404 (23:26-23:58)
- **根因**: Nginx conf.d/soa.laziestlife.com.conf（HTTPS server块）中缺少 `/admin2/` location
- HTTP走 sites-available/iot-doll (root=/var/www/iot-ai-doll/frontend) → 能找到文件 → 200
- HTTPS走 conf.d (root=/var/www/iot-ai-doll/frontend/dist) → 找不到admin2 → 404
- **修复**: 在conf.d的HTTPS server块中添加 `/admin2/` location，root指向 `/var/www/iot-ai-doll/frontend`

### 阶段2: 页面加载中... 数据不显示 (23:58-00:04)
- **根因**: 前端API路径 `/api/admin/models` 与后端不匹配
- 后端实际路由是 `/api/epet/models`（在 epet.js 中注册）
- **修复**:
  - CompanionList.tsx: API路径改为 `/api/epet/models`，数据解析从 `data.data` 改为 `data.models`
  - CompanionEdit.tsx: 同步修正API路径
- 本地构建后SCP上传到服务器

### 关键配置备忘
| 配置项 | 值 |
|--------|-----|
| Vite base | `/admin2/` |
| React Router basename | `/admin2` |
| 服务器文件路径 | `/var/www/iot-ai-doll/frontend/admin2/` |
| Nginx HTTP server | sites-available/iot-doll |
| Nginx HTTPS server | conf.d/soa.laziestlife.com.conf |
| 后端API路由 | `/api/epet/models` (不是 /api/admin/models) |

### 经验教训
1. Nginx配置分散在多个文件（conf.d + sites-enabled），HTTP和HTTPS可能走不同server块
2. 前端API路径必须与后端index.js注册的路由完全一致
3. conf.d文件可能被覆盖，修改后要验证持久性
4. 后端路由清单：/api/db, /api/admin/rag-kbs, /api/epet/*, /api/game-assets 等

## Admin2 "页面加载中..." 真正根因修复 (00:04-00:07)

### 问题
阶段2修复后用户反馈**仍然显示"页面加载中..."**，Console无报错。

### 真正根因（三层问题叠加）

#### 第1层（致命）：App.tsx 使用硬编码空壳组件
```tsx
// App.tsx 原代码 — 完全没有 import 真正的页面组件！
const CompanionsPage: React.FC = () => 
  <div><h1>宠物型号管理</h1><p>页面加载中...</p></div>;
const RAGPage: React.FC = () => 
  <div><h1>RAG 知识库管理</h1><p>页面加载中...</p></div>;
```
**之前修改 CompanionList.tsx 的 API 路径完全无效，因为根组件根本没有引用它。**
通过 `grep -o 'Xn=.*' index-BIaaBC89.js` 确认构建产物确实是硬编码空壳。

#### 第2层：API 路径不匹配（已修但未生效）
CompanionList.tsx 中 `/api/admin/models` → 应为 `/api/epet/models`

#### 第3层：RAGList.tsx import 报错
`RAGKnowledgeBase` interface 从 `rag.ts` import 时 Vite/Rolldown 报 `MISSING_EXPORT` 错误。
**原因推测**: Rolldown (Vite 新打包器) 对 interface export 可能有 bug，或与同目录 `index.ts` 存在模块解析冲突。
**解决方案**: 将 interface 内联定义到 RAGList.tsx，不再外部 import。同时删除冗余的 `api/index.ts`。

### 最终修复操作
1. **App.tsx**: 删除硬编码空壳，改为 `import CompanionList from './pages/companions/CompanionList'`
2. **CompanionList.tsx**: API路径 `/api/admin/models` → `/api/epet/models`
3. **RAGList.tsx**: interface RAGKnowledgeBase 内联定义
4. **删除 api/index.ts**: 旧的重复 API 文件（与 rag.ts / companions.ts 冲突）

### 验证结果
- 新 JS 文件名: `index-B9lBmn4-.js`（旧版 `index-BIaaBC89.js`，hash 变化确认代码更新）
- `grep 'api/epet/models'` 在新 JS 中找到匹配 ✅
- SCP 部署成功

### 核心经验教训
1. **修改源码后必须确认组件被实际引用** — 空壳/占位组件是最难发现的 bug，Console 不报错但功能完全失效
2. **Vite content hash 变化 = 代码确实进入构建** — 如果 hash 不变说明修改没生效（缓存/没引用）
3. **Rolldown (Vite 6 新打包器) 可能有 interface export 的 bug** — 遇到 MISSING_EXPORT 尝试内联 type
4. **调试方法**: 直接 grep 构建产物的 minified 代码确认组件是否为空壳

## Admin2 编辑页面空白修复 (00:50-00:53)

### 问题
用户访问 `/admin2/admin/companions/1/edit` 显示空白页面。

### 根因（两层问题）

#### 第1层：App.tsx 缺少编辑页面路由
App.tsx 只定义了 `/companions` 路由，**没有定义 `/companions/:id/edit` 路由**，也没有 import CompanionEdit 组件。

#### 第2层：导航路径多了 `/admin/` 前缀
CompanionList.tsx 中的编辑按钮导航路径是：
```tsx
navigate(`/admin/companions/${companion.model_id}/edit`)
```
但 React Router 的 basename 已经是 `/admin2`，所以实际 URL 变成 `/admin2/admin/companions/...`。

### 修复操作
1. **App.tsx**: 添加 `/companions/:id/edit` 路由并 import CompanionEdit 组件
2. **CompanionList.tsx**: 导航路径从 `/admin/companions/...` 改为 `/companions/...`（去掉 `/admin` 前缀）
3. **CompanionList.tsx**: 修复语法错误（缺少闭合的 `)}`）

### 验证结果
- 新 JS 文件名: `index-CUgV9WQi.js`
- SCP 部署成功 ✅

### 正确访问路径
- 列表页: `https://soa.laziestlife.com/admin2/companions`
- 编辑页: `https://soa.laziestlife.com/admin2/companions/1/edit`
- **注意**: URL 中不要包含 `/admin/` 中间段

### 经验教训
1. **React Router basename 和导航路径不要重复** — basename="/admin2" 时，navigate 路径应该是 "/companions/..." 而非 "/admin2/companions/..." 或 "/admin/companions/..."
2. **添加新页面时要同步更新路由表** — 有导航链接但没定义路由 = 空白页面
3. **JSX 语法错误（缺少闭合括号）会导致整个构建失败** — 开发时要留意编译器的红色错误提示

4. fetch 的 response.json() 加 try-catch 保护

#### RAGEdit.tsx
1. **完全重写**，不再 import rag.ts 的任何函数
2. 直接使用 fetch API，所有 json() 调用加 try-catch

## Admin2 /rag/new 显示"加载中"修复 (01:01-01:06)

### 问题
访问 `/admin2/rag/new` 页面空白，只显示"加载中..."文字，Console 无报错。

### 根因：React Router v6 路由顺序错误
App.tsx 中路由定义顺序为：
```
/rag          → RAGList      ← 先匹配了这里！
/rag/new      → RAGEdit      ← 永远不会到达
/rag/:id/edit → RAGEdit      ← 永远不会到达
```
React Router v6 按 Routes 定义顺序匹配，`/rag` 作为前缀先捕获了 `/rag/new`，
导致渲染的是 RAGList（其 loading 状态显示"加载中..."）而非 RAGEdit。

### 修复
调整 App.tsx 路由顺序：将更具体的路径（`/rag/new`, `/rag/:id/edit`）放到通用路径（`/rag`）**前面**。

### 验证结果
- 新 JS 文件名: `index-Ci08jn08.js`
- SCP 部署成功 ✅

### 经验教训
**React Router v6 路由顺序至关重要** — 具体路径必须放在通用路径前面，否则会被提前拦截匹配。这是 SPA 路由的经典坑。
