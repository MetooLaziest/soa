## 技术总结

### 学到的经验

1. **浏览器缓存是集成的常见坑**
   - 即使服务器端已更新，浏览器可能显示旧版
   - 必须考虑缓存破坏策略（版本号、时间戳、ETag）

2. **SVG 集成的最佳实践**
   - 使用 JavaScript 动态切换 `src` 属性
   - 避免不必要的重复加载（检查当前 src 再决定更不更新）
   - 添加过渡动画提升用户体验

3. **expect + scp 是可靠的自动化上传方案**
   - 适合密码已知的场景
   - 比 SSH key 配置更简单（但安全性较低）
   - 注意密码中的特殊字符需要转义

4. **状态判断需要优先级**
   - dead > hungry > happy > poop > sleeping > idle
   - 避免多个状态同时触发导致闪烁

#### 问题 1：喂食后饱腹感从 0 → 100（不符合预期）

**原因分析**：
- 数据库字段 `hunger` 初始值为 100（错误）
- `calcFullness()` 函数逻辑错误：如果没有喂食记录 (`lastFeedAt` 为 NULL)，返回 100

**修复方案** (`epet-router.mjs`):
```javascript
// ✅ 修复后的 calcFullness 函数
function calcFullness(lastFeedAt, hunger) {
  // 如果从来没有喂过食，饱腹感为 0
  if (!lastFeedAt || !hunger) return 0;  // ✅ 修复：返回 0，不是 100
  
  const now = new Date();
  const last = new Date(lastFeedAt);
  const diffMs = now - last;
  const diffMinutes = diffMs / (1000 * 60);
  
  // 每10分钟衰减 (100/144) ≈ 0.694
  const decay = (diffMinutes / 10) * (100 / 144);
  const fullness = Math.max(0, hunger - decay);
  
  return Math.round(fullness * 100) / 100;
}
```

**同时修复**：
- 创建宠物时 `hunger=0`（不是 100）
- `last_feed_at` 初始为 NULL
- 喂食时 `hunger = MIN(100, hunger + 35)`
