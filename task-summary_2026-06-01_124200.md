## 目标
修复3个bug + 添加2个新按钮

## 问题 & 修复

### 1. 白色玩偶图片不显示
- **根因**: 前端用 `pet-${monsterType}.png` 拼接路径，white→pet-white.png 不存在
- **nginx行为**: 文件不存在时返回 index.html (text/html)，img 标签无法渲染
- **修复**: `getImagePath()` 函数做特殊映射: white → `/epet/pet-idle.png`
- **教训**: pet-green.png(之前SVG占位图幻觉)已删除，但代码路径未同步修正

### 2. 两只宠物重叠
- **根因**: 只有一个 `.pet-img-wrap` 容器，无法容纳两张图片
- **修复**: 重构为 `.pet-card` 多卡片系统:
  - 每只宠物独立 card: name-tag + img-wrap(140x140) + mini-stats
  - flex布局 + margin-left 间距
  - collection-btn 用 absolute 定位在右上角

### 3. 宠物名称更新
- 数据库 UPDATE: 9527=团子糯糯, 2333=海浪沫沫, 10086=糖心莓莓

### 4. 新增按钮
- 【旅行明信片】📮 紫色渐变 → toast "功能还在开发中 💌"
- 【购买新产品】🛍️ 粉红渐变 → toast "功能还在开发中 🛒"
- 位于 bottom-row (打开小程序/绑定智能家居) 下方

## 技术细节
- `visiblePets[]` 数组替代单 `pet` 对象，支持多宠物状态
- `renderPetCards()` 动态生成 N 个 .pet-card DOM
- 每个 card 独立状态判断(walking/breathing/sleeping filter)
- API auto-refresh 改为 loadVisiblePets() 每15秒
- feed/pet/clean 操作仍针对 currentPetId (第一只展示的)

## v6.1 补丁 - 12:48
### Bug: 团子糯糯默认强制展示
- **根因**: `loadVisiblePets()` 总是把 `currentPetId`(URL参数/默认9527)作为第一只加载，不管藏品库是否隐藏
- **修复**: 改为只从 `monsters` 列表中取 `isVisible=true` 的宠物来加载；若当前ID不在可见列表中则自动切换primary
- **效果**: 藏品库选谁就显示谁，最多2只，不会多出
